```
resource "yandex_mdb_mysql_cluster" "main" {

  environment = "PRESTABLE"

  version     = var.mysql_version

      zone      = local.private_subnets[host.value].zone
      subnet_id = yandex_vpc_subnet.private[host.value].id

  }
```
```terraform
}
```

 - Создать региональный мастер Kubernetes с размещением нод в трёх разных подсетях.
Региональный мастер в зонах a, b, d, каждая в своей public-подсети. У мастера включён публичный IP.
k8s.tf:
```terraform
resource "yandex_kubernetes_cluster" "main" {
  name        = local.k8s_cluster_name
  network_id  = yandex_vpc_network.main.id
  // ...
  master {
    regional {
      region = "ru-central1"

      dynamic "location" {
        for_each = local.k8s_zones
        content {
          zone      = local.public_subnets[location.value].zone
          subnet_id = yandex_vpc_subnet.public[location.value].id
        }
      }
    }

    version   = var.k8s_version
    public_ip = true
  }
}
```

Зоны перечислены в locals.tf:
```terraform
  k8s_zones = ["a", "b", "d"]
```

 - Создать группу узлов, состояющую из трёх машин с автомасштабированием до шести.
variables.tf:
```terraform
variable "k8s_node_min" {
  type    = number
  default = 3
}

variable "k8s_node_max" {
  type    = number
  default = 6
}

variable "k8s_node_initial" {
  type    = number
  default = 3
}
```

Группа узлов в k8s.tf:
```terraform
resource "yandex_kubernetes_node_group" "main" {
  cluster_id = yandex_kubernetes_cluster.main.id
  name       = local.k8s_node_group_name
  version    = var.k8s_version

  instance_template {
    // ноды в public-a, NAT, security group k8s
    network_interface {
      nat                = true
      subnet_ids         = [yandex_vpc_subnet.public["a"].id]
      security_group_ids = [yandex_vpc_security_group.k8s.id]
    }
    // ...
  }

  scale_policy {
    auto_scale {
      min     = var.k8s_node_min
      max     = var.k8s_node_max
      initial = var.k8s_node_initial
    }
  }

  allocation_policy {
    location {
      zone = local.public_subnets["a"].zone
    }
  }
}
```
при auto_scale в YC allocation_policy допускает только одну зону — поэтому ноды в ru-central1-a, а отказоустойчивость мастера обеспечивается тремя зонами в master.regional

 - *Создать сервис-типы Load Balancer и подключиться к phpmyadmin. Предоставить скриншот с публичным адресом и подключением к БД.

phpmyadmin-service.yaml:
```yaml
apiVersion: v1
kind: Service
metadata:
  name: phpmyadmin
  labels:
    app: phpmyadmin
spec:
  type: LoadBalancer
  selector:
    app: phpmyadmin
  ports:
    - name: http
      port: 80
      targetPort: 80
```

```bash
kubectl apply -f k8s/phpmyadmin-service.yaml
kubectl get svc phpmyadmin
```
