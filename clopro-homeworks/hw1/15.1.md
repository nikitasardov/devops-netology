### Задание 1. Yandex Cloud 

**Что нужно сделать**

1. Создать пустую VPC. Выбрать зону.
network.tf:
```terraform
resource "yandex_vpc_network" "main" {
  name = var.vpc_name
}
```

2. Публичная подсеть.
 - Создать в VPC subnet с названием public, сетью 192.168.10.0/24.
network.tf:
```terraform
resource "yandex_vpc_subnet" "public" {
  name           = var.public_subnet_name
  zone           = var.default_zone
  network_id     = yandex_vpc_network.main.id
  v4_cidr_blocks = var.public_cidr
}
```
variables.tf:
```terraform
variable "public_subnet_name" {
  type    = string
  default = "public"
}
variable "public_cidr" {
  type        = list(string)
  default     = ["192.168.10.0/24"]
  description = "CIDR публичной подсети"
}
```

 - Создать в этой подсети NAT-инстанс, присвоив ему адрес 192.168.10.254. В качестве image_id использовать fd80mrhj8fl2oe87o4e1.
compute.tf:
```terraform
resource "yandex_compute_instance" "nat" {
  name        = local.nat_instance_name
  platform_id = var.vm_platform_id
  zone        = var.default_zone
}
...
```
security.tf:
```
resource "yandex_vpc_security_group" "nat" {
  name       = local.sg_nat_name
  network_id = yandex_vpc_network.main.id

  ingress {
    protocol       = "TCP"
    description    = "ssh"
    v4_cidr_blocks = ["0.0.0.0/0"]
    port           = 22
  }

  egress {
    protocol       = "ANY"
    description    = "any"
    v4_cidr_blocks = ["0.0.0.0/0"]
    from_port      = 0
    to_port        = 65535
  }
}
```

 - Создать в этой публичной подсети виртуалку с публичным IP, подключиться к ней и убедиться, что есть доступ к интернету.
compute.tf:
```
resource "yandex_compute_instance" "public" {
  name        = local.public_vm_name
  // ...
  boot_disk {
    initialize_params {
      image_id = data.yandex_compute_image.ubuntu.image_id
      // ...
    }
  }
  network_interface {
    subnet_id          = yandex_vpc_subnet.public.id
    nat                = true
    security_group_ids = [yandex_vpc_security_group.public.id]
  }
  metadata = {
    serial-port-enable = 1
    ssh-keys           = "${var.vm_user}:${var.ssh_public_key}"
  }
}
```

3. Приватная подсеть.
 - Создать в VPC subnet с названием private, сетью 192.168.20.0/24.
network.tf:
```terraform
resource "yandex_vpc_subnet" "private" {
  name           = var.private_subnet_name
  zone           = var.default_zone
  network_id     = yandex_vpc_network.main.id
  v4_cidr_blocks = var.private_cidr
  route_table_id = yandex_vpc_route_table.private.id
}
```
Отличие от public-подсети route_table_id. Исходящий трафик из private пойдёт не напрямую в интернет, а через таблицу маршрутизации.

 - Создать route table. Добавить статический маршрут, направляющий весь исходящий трафик private сети в NAT-инстанс.
network.tf:
```terraform
resource "yandex_vpc_route_table" "private" {
  name       = local.route_table_name
  network_id = yandex_vpc_network.main.id

  static_route {
    destination_prefix = "0.0.0.0/0"
    next_hop_address   = var.nat_instance_ip
  }
}
```
Адрес NAT задан через переменную в variables.tf:
```terraform
variable "nat_instance_ip" {
  type    = string
  default = "192.168.10.254"
}
```

 - Создать в этой приватной подсети виртуалку с внутренним IP, подключиться к ней через виртуалку, созданную ранее, и убедиться, что есть доступ к интернету.

compute.tf:
```
resource "yandex_compute_instance" "private" {
  name        = local.private_vm_name
  platform_id = var.vm_platform_id
  zone        = var.default_zone
  // ...
  network_interface {
    subnet_id          = yandex_vpc_subnet.private.id
    nat                = false
    security_group_ids = [yandex_vpc_security_group.private.id]
  }
  metadata = {
    serial-port-enable = 1
    ssh-keys           = "${var.vm_user}:${var.ssh_public_key}"
  }
}
```
