Группа ВМ создаёт target group через блок load_balancer. instance_group.tf:
```terraform
  load_balancer {
    target_group_name        = local.lamp_tg_name
    target_group_description = "NLB target group for ${local.lamp_ig_name}"
  }
```
locals.tf:
```terraform
  lamp_tg_name = "${local.project}-${local.env}-lamp-tg"
  nlb_name     = "${local.project}-${local.env}-nlb"
```
Сетевой балансировщик с публичным IP и health check.
load_balancer.tf:
```terraform
resource "yandex_lb_network_load_balancer" "lamp" {
  name      = local.nlb_name
  folder_id = var.folder_id

  listener {
    name        = "http"
    port        = 80
    target_port = 80
    external_address_spec {
      ip_version = "ipv4"
    }
  }

  attached_target_group {
    target_group_id = yandex_compute_instance_group.lamp.load_balancer[0].target_group_id

    healthcheck {
      name = "http"
      http_options {
        port = 80
        path = "/"
      }
    }
  }
}
```
NLB сохраняет IP клиента, на бэкендах HTTP :80 нужно разрешить из `0.0.0.0/0` (не только из CIDR подсети). security.tf:
```terraform
  ingress {
    protocol       = "TCP"
    description    = "http from internet via nlb"
    v4_cidr_blocks = ["0.0.0.0/0"]
    port           = 80
  }
```
outputs.tf:
```terraform
output "load_balancer" {
  value = {
    name        = yandex_lb_network_load_balancer.lamp.name
    external_ip = local.nlb_external_ip
    url         = "http://${local.nlb_external_ip}/"
  }
}
```
