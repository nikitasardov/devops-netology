# Домашнее задание к занятию 11 «Teamcity»

## Подготовка к выполнению

1. В Yandex Cloud создайте новый инстанс (4CPU4RAM) на основе образа `jetbrains/teamcity-server`.
2. Дождитесь запуска teamcity, выполните первоначальную настройку.
![](https://i.postimg.cc/fRFfpq7h/2025-10-23-15-09.png)
3. Создайте ещё один инстанс (2CPU4RAM) на основе образа `jetbrains/teamcity-agent`. Пропишите к нему переменную окружения `SERVER_URL: "http://<teamcity_url>:8111"`.
![](https://i.postimg.cc/k49qCmWP/2025-10-23-15-18.png)
4. Авторизуйте агент.
![](https://i.postimg.cc/KjtZh8Hk/2025-10-23-15-30.png)
5. Сделайте fork [репозитория](https://github.com/aragastmatb/example-teamcity).
![](https://i.postimg.cc/m2cx0DK3/2025-10-23-15-20.png)
6. Создайте VM (2CPU4RAM) и запустите [playbook](./infrastructure).

## Основная часть

1. Создайте новый проект в teamcity на основе fork.
2. Сделайте autodetect конфигурации.
![](https://i.postimg.cc/BtWX2zdv/2025-10-23-15-37.png)
3. Сохраните необходимые шаги, запустите первую сборку master.
![](https://i.postimg.cc/8cFsxGhK/2025-10-23-15-57.png)
4. Поменяйте условия сборки: если сборка по ветке `master`, то должен происходит `mvn clean deploy`, иначе `mvn clean test`.
5. Для deploy будет необходимо загрузить [settings.xml](./teamcity/settings.xml) в набор конфигураций maven у teamcity, предварительно записав туда креды для подключения к nexus.
6. В pom.xml необходимо поменять ссылки на репозиторий и nexus.
7. Запустите сборку по master, убедитесь, что всё прошло успешно и артефакт появился в nexus.
![](https://i.postimg.cc/25TC751g/2025-10-23-16-37.png)
8. Мигрируйте `build configuration` в репозиторий.
9.  Создайте отдельную ветку `feature/add_reply` в репозитории.
10. Напишите новый метод для класса Welcomer: метод должен возвращать произвольную реплику, содержащую слово `hunter`.
11. Дополните тест для нового метода на поиск слова `hunter` в новой реплике.
12. Сделайте push всех изменений в новую ветку репозитория.
13. Убедитесь, что сборка самостоятельно запустилась, тесты прошли успешно.
![](https://i.postimg.cc/dVvwyCWJ/2025-10-23-17-26.png)
14. Внесите изменения из произвольной ветки `feature/add_reply` в `master` через `Merge`.
15. Убедитесь, что нет собранного артефакта в сборке по ветке `master`.
16. Настройте конфигурацию так, чтобы она собирала `.jar` в артефакты сборки.
17. Проведите повторную сборку мастера, убедитесь, что сбора прошла успешно и артефакты собраны.
![](https://i.postimg.cc/RZ3yqZmM/2025-10-23-17-36.png)
18. Проверьте, что конфигурация в репозитории содержит все настройки конфигурации из teamcity.
![](https://i.postimg.cc/yYXc9wmp/2025-10-23-18-02.png)
![](https://i.postimg.cc/cCW6LLb6/2025-10-23-18-03.png)
![](https://i.postimg.cc/y8xxd5vj/2025-10-23-18-02-1.png)
19. В ответе пришлите ссылку на репозиторий.
