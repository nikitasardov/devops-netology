1. Вас пригласили настроить мониторинг на проект. На онбординге вам рассказали, что проект представляет из себя платформу для вычислений с выдачей текстовых отчетов, которые сохраняются на диск. Взаимодействие с платформой осуществляется по протоколу http. Также вам отметили, что вычисления загружают ЦПУ. Какой минимальный набор метрик вы выведите в мониторинг и почему?

ОТВЕТ: в качестве минимального набора метрик предлагаю:
- CPU: загрузка
  Необходимо, т.к. по условию платформа производит вычисления, а CPU - критический ресурс для вычислений. Load Average для минимального набора может быть избыточен, т.к. эта метрика показывает общую нагрузку системы (включая ожидание I/O), но так как мы мониторим диск отдельно (IO wait) и память, то достаточно загрузки CPU.
- Диск: свободное место, inodes, IO wait
  Платформа пишет результаты вычислений в текстовые файлы на диск - необходимо контролировать основные параметры диска.
- HTTP: время отклика, количество запросов, доля ошибок (4xx/5xx)
  Время отклика и доля ошибок для контроля качества обслуживания, количество запросов для понимания нагрузки.
- RAM: использование памяти (для минимального набора можно без SWAP)
  Система работает с вычислениями, необходимо обеспечить достаточный объем свободной памяти для хранения данных во время вычислений.

По качеству обслуживания предложим показывать:
- uptime или availability - процент времени, когда сервис доступен, вместо технических метрик (CPU/RAM).

- активность пользователей - нагрузка, количество запросов в единицу времени
- речь уже не о минимальном наборе метрик, можем предложить вычислять процент запросов, обработанных за определенное время (пример: 95% запросов < 1 сек)
- процент успешно сохраненных отчетов (дисковые метрики) (вероятно, потребуется доработка приложения, чтобы определять событие успеха)
- процент успешно выполненных вычислений (метрика CPU) (вероятно, потребуется доработка приложения, чтобы определять событие успеха)

9. Изучите список [telegraf inputs](https://github.com/influxdata/telegraf/tree/master/plugins/inputs). 
Добавьте в конфигурацию telegraf следующий плагин - [docker](https://github.com/influxdata/telegraf/tree/master/plugins/inputs/docker):
```
[[inputs.docker]]
  endpoint = "unix:///var/run/docker.sock"
```

Дополнительно вам может потребоваться донастройка контейнера telegraf в `docker-compose.yml` дополнительного volume и 
режима privileged:
```
  telegraf:
    image: telegraf:1.4.0
    privileged: true
    volumes:
      - ./etc/telegraf.conf:/etc/telegraf/telegraf.conf:Z
      - /var/run/docker.sock:/var/run/docker.sock:Z
    links:
      - influxdb
    ports:
      - "8092:8092/udp"
      - "8094:8094"
      - "8125:8125/udp"
```

После настройке перезапустите telegraf, обновите веб интерфейс и приведите скриншотом список `measurments` в веб-интерфейсе базы telegraf.autogen . Там должны появиться метрики, связанные с docker.
